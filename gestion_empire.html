Présentation

Gestion empire est un jeu web multijoueur de gestion de ressources où le joueur gère un empire qu'il doit faire progresser.

Maintenance

Pour cette maintenance, j'ai choisi d'ajouter une map qui qui permet de positionner les joueurs dans le monde. Elle permet de d'attaquer un joueur en cliquant sur sa position.

Veille Technologique

Langage côté serveur

Nous avons choisi PHP par rapport aux alternatives qu'il pouvait y avoir comme Python avec Django ou Ruby avec Ruby on Rails car le projet se déroulant sur une période courte, nous avons choisi le PHP que nous connaissions déjà et qui a une documentation fourni car étant un langage étant très utilisé. Nous avons choisi de ne pas prendre de framework car même si nous connaissions déjà PHP nous ne le maitrisons pas assez. Étant quand même dans une démarche d'apprentissage, le PHP pur nous paraissait plus pertinent car ca nous permet d'apprendre le langage, ses particularités, et d'avoir des bases solides en PHP ce qui me parait nécessaire dans l'utilisation et l'apprentissage d'un framework.

Stockage de données

Nous avons choisi de stocker nos données dans une base de données relationnelle car étant un jeu multijoueur, il faudra stocker beaucoup de données et elles seront souvent liées. Les alternatives comme les fichiers JSON, XML ne sont donc pas une bonne idée.

Autre

Nous avons utilisé l'AJAX pour faire des requêtes non bloquante au serveur et donner une expérience de jeu agréable à l'utilisateur. Il y a des alternatives comme les WebSocket cependant le professeur qui nous a donné ce projet universitaire nous avait donné la consigne d'utiliser AJAX obligatoirement.
